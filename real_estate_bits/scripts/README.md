## üîç Soluci√≥n de Problemas

### Error: "Autenticaci√≥n fallida"

**Causa:** Credenciales incorrectas o base de datos no existe.

**Soluci√≥n:**
```python

    'url': 'https://...',  # Sin barra al final
    'db': 'nombre-exacto-base-datos',
    'username': 'admin',
    'password': 'contrase√±a-correcta'
}
```

### Error: "No se pudo crear la propiedad"

**Causas comunes:**
1. Faltan campos requeridos
2. Relaciones inv√°lidas (ciudad, departamento no existen)
3. Restricciones SQL (c√≥digos duplicados)

**Soluci√≥n:**
```python

# Verificar que existan registros base
# - Pa√≠s Colombia
# - Departamento
# - Ciudad
```

### Error: "No se encontr√≥ regi√≥n/barrio"

**Causa:** La regi√≥n no existe en la base de datos.

**Soluci√≥n:** El script crear√° autom√°ticamente la regi√≥n si no existe. Verificar que `state_name` y `city_name` sean correctos.

### Advertencia: "No se pudo crear direcci√≥n de facturaci√≥n"

**Causa:** Normalmente no cr√≠tico. El partner principal no tiene ID v√°lido.

**Soluci√≥n:** Verificar que `partner_data` tenga `name` m√≠nimo.

### Performance Lento

**Causa:** Muchas propiedades, conexi√≥n lenta.

**Soluciones:**
1. Reducir cantidad de propiedades por lote
2. Usar conexi√≥n local si es posible
3. Comentar logs DEBUG:
   ```python
   logger.setLevel(logging.INFO)
   ```
